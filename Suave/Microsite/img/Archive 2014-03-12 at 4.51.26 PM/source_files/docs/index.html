2 Quick usagetop

Plugin usage is really simple, the first thing you need to do is to load jQuery and the plugin code by adding the following script line in your page head section
<script type="text/javascript" src="js/jquery-1.7.min.js"></script>
<script type="text/javascript" src="js/pe.shiner/jquery.pixelentity.shiner.min.js"></script>
If you are already using jQuery in your page, omit the first script line in the above block and make sure you add the plugin code after jQuery. This plugin has been designed to work with version 1.6 or 1.7 of jQuery.
If you are using an older version of jQuery then that stated above, the plugin may not function correctly.

Next,add the images/logos to which you wish to add the shiner effect. Each image will need to be wrapped in a div with a class of peShiner like shown here below. This example also shows an image with a link, but the link is optional and can be omitted if not needed.
<div class="peShiner">
     <a href="yourpage.html"><img src="img/logo.png" /></a>
</div>
Now all that is left is to activate the plugin by adding the following script tag into the head section of the containing HTML page.
<script>
    jQuery(document).ready(function () {
        $(".peShiner").peShiner();
    })
</script>

3 Mouse Over Modetop

The effect can be configured so that it only appears when the user rolls over the image/logo with their mouse. This mode can be activated by setting the hover configuration option value to true. Once the plugin has been activated and setup (See the previous section), to enable this hover mode just make the following change to the script which was added to the head section of the HTML page:
$(".peShiner").peShiner({hover:true});
This is an example of how you can alter global defaults for the plugin (apply to all images in the HTML page which are using the shiner efvect). The hover setting is just one of many available parameters, we'll cover all of these in the next section.

4 Effect Customizationtop

4.1 Global Options

As mentioned in the previous section, the Shiner plugin's behaviour can be customized globally by setting parameters when it is activated in the head section of the HTML page. Such a plugin call, while setting all global configuration values would look like this:
$(".peShiner").peShiner({
    hover: false,
    duration: 2,
    angle: 15,
    size: 100,
    color: "monoHL",
    glow: 2,
    repeat: 0.5,
    reverse: false,
    delay: 0
});
You're not forced to set all the options, if a a parameter is not set by you, it will keep its default value. See the table below for an explanation of these parameters and their default values, where applicable.

4.2 Configuration options

Setting glow to 1-2 will also add a 1-2px margin to the image in order to make it visible, this will need to be taken into account in your page layout so that the plugin doesn't interfere with your layout. If this additional margin is having undesidered effects on the page layout, just set this parameter to 0

4.3 Per Image Options


You can apply the Shiner effect to multiple elements in a page at the same time. In the previous section we looked at how to change parameters globally, which affect all elements in the page which have the effect applied. However there are cases in which you would want to control the parameters of the effect on an image by image basis. This can be done by setting options directly in the individual elment's markup using custom data attributes. By using these settings each image may have a different configuration of the Shiner effect applied to it as shown here below:
<!--image 1-->


<div class="peShiner">
    <img src="img/logo1.png" data-color="limeHL" data-glow="0" />
</div>




<!--image 2-->


<div class="peShiner">
    <img src="img/logo2.png" data-color="purpleHL" data-angle="45"/>
</div>
In the above example we have 2 logos, the first one will use a limeHL color grandient with a glow value of 0. The second logo will use a purpleHL gradient and have a Shiner bar angle of 45 degrees. Data attributes may be added to specify all of the parameters which are available in the global parameters section (Refer to that section above for a list and description of each parameter).
Data attributes are formed by simply prefixing any of the global parameters with data- and then setting the parameter's value inside of inverted commas just like a normal HTML attribute, like this data-reverse="true"

5 Advanced Usetop

5.1 Creating a Layered Composition Using CSS


In the download package the file example04.html contains an example of using the Shiner effect item as part of a layered composition of graphical elements, like what is seen in the iTunes logo example in the online preview. Basically what this means is that image layers are stacked on top of each other so that it looks like one complete image, while the shiner effect gets applied to only one layer, in this case, the colored background layer. This composition is made up of 3 layers.

  1. Top Layer - logo.
  2. Middle Layer - logo coloured background with Shiner effect applied
  3. Bottom Layer - frame or sourround.
The markup for this layered composiotion looks like this:
<div class="composition">
                <!--layer 3 - top-->
                <img src="img/example04/layer3.png" alt="" class="layer3" />


                <!--layer 2 - middle - shiner effect-->
                <div class="peShiner">
                    <img src="img/example04/layer2.png" data-color="blueHL" /></a>
                </div>


                <!--later 3 - bottom-->
                <img src="img/example04/layer1.png" alt="" class="layer1" />
            </div>
To make sure all of the elements are aligned correctly, each image should be exactly the same size and the elements making up the layer should be absolutely positioned using CSS. In the example provided in the download package, the image size is 200x200px and the CSS to correctly stack the elements looks like this:
/*set width , height & position*/


.composition, .composition > img, .peShiner img
            {
                width:200px;
                height:200px;
                position:absolute;
                top:0;
                left:0;
            }


/*set z-index values so items will stack correctly*/


img.layer1{
                z-index:1;
            }


.peShiner{
                z-index:2;
            }


img.layer3{
                z-index:3;
            }

5.2 Plugin API


The plugin comes with very simple API that allows it to control the bahaviour of the plugin using external javascript, which may be needed in cases of advanced use. An example of such use might be; Showing the effect in response to a user's actions on another active element in your page.
To control the Shiner effect's behaviour, the first step is to get the API object:
var api = $(".peShiner").peShiner({api:true}));
By setting "api" to "true", the plugin will return its API object instead of the standard jQuery object. You can then use this API object as follows:
// pause the effect
api.pause();


// resume the effect
api.resume();
if pause(); is called when the effect is already running, the current transition will be completed before the pause takes effect
The Shiner effect can also be activated, in the head section of the HTML page, in a paused state like this:
var api = $(".peShiner").peShiner({api:true, paused:true}));

5.3 Using API With Multiple Effect Instances


If you have the plugin applied to multiple elements within the same page, you must get the API objects one at a time for these API calls to function correctly. To do this we recommend giving each Shiner element a unique ID like this:
<!--element 1-->


<div class="peShiner" id="shiner1">
        <img src="img/logo1.png" data-color="limeHL" />
</div>


<!--element 2-->


<div class="peShiner" id="shiner2">
        <img src="img/logo2.png" data-color="purpleHL" />
</div>
Then access the API object like this:
var api1 = $("#shiner1").peShiner({api:true, paused:true}));


var api2 = $("#shiner2").peShiner({api:true, paused:true}));
